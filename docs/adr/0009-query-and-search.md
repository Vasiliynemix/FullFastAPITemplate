# 0009. Универсальные фильтрация/сортировка/поиск + умный (fuzzy) поиск

## Status
Accepted

## Context
Списочным ручкам нужны фильтры по полям, сортировка и поиск. Хотелось НЕ перечислять
поля под каждую модель руками, не плодить раздутые query-схемы и не открыть дыры
(SQL-инъекции, фильтр по произвольному имени). Плюс — «умный» поиск, терпящий опечатки
(«всая» → «вася»), который на чистом ORM/ILIKE не делается.

## Decision
1. **Универсальный слой** `app/db/query.py` + `BaseRepository.paginate()`:
   * фильтры `?field__op=value` (eq/ne/gt/ge/lt/le/like/ilike/contains/in), оператор по
     умолчанию `eq`;
   * сортировка `?sort=field` / `?sort=-field`, дефолт `created_at` asc;
   * поиск `?q=...`. Работает с ЛЮБОЙ моделью через интроспекцию колонок.
2. **Безопасность**: имена полей валидируются по реальным колонкам (белый список →
   неизвестное поле = `400`); операторы — из фиксированного списка; значения
   параметризуются (нет SQL-инъекций) и приводятся к типу колонки; `q` санитизируется
   (срезаются HTML-теги/control-символы).
3. **Умный поиск** — триграммы Postgres (`pg_trgm`): `word_similarity(q, поле)` находит
   похожее СЛОВО внутри значения (терпит опечатки даже в многословных полях), сортировка
   по релевантности. Порог — `SEARCH_SIMILARITY_THRESHOLD`. На не-Postgres — фолбэк ILIKE.
4. **Автоматизация индексов**: GIN-триграммные индексы объявляются декларативно
   (`trgm_index()` в `__table_args__`), расширение `pg_trgm` создаётся само (DDL-event для
   create_all, `ensure_pg_trgm()` для миграций) — без сырого SQL (см. [ADR-0007] про связи
   и app/db/ddl.py).

## Consequences
- (+) Новой списочной ручке достаточно `ListParamsDep` + (для поиска) `search_fields` и
  один `trgm_index` — без перечисления полей и без raw SQL.
- (+) Безопасно: инъекции по именам/значениям исключены интроспекцией и параметризацией.
- (+) Опечатки реально ловятся (проверено вживую: «всая» → «Вася Пупкин»).
- (−) Динамические фильтры `field__op` не видны типами в Swagger (любое поле) — их синтаксис
  вынесен в общее описание API сверху, а не дублируется в каждой ручке.
- (−) Низкий порог похожести терпим к опечаткам, но пропускает слабые совпадения; они идут
  в конце (сортировка по релевантности), порог настраивается.
- (−) `--autogenerate` неточно распознаёт `gin_trgm_ops` — миграции по trgm-индексам ревьюим
  или пишем через `create_trgm_index`.
