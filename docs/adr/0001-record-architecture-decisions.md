# 0001. Вести Architecture Decision Records

## Status
Accepted

## Context
Шаблон содержит много неочевидных решений (паттерны, флаги, компромиссы под нагрузку).
Код показывает «как», но не «почему». Без фиксации мотивации решения со временем
выглядят произвольными, их боятся менять или повторяют ошибки, от которых уже ушли.

## Decision
Ведём ADR в `docs/adr/`. Каждое значимое архитектурное решение — отдельный файл по
формату Status / Context / Decision / Consequences. ADR неизменяемы: пересмотр решения
оформляется новым ADR, который помечает старый как superseded.

## Consequences
- (+) Сохраняется контекст: почему выбран подход и какие альтернативы отвергнуты.
- (+) Онбординг и ревью быстрее — мотивация под рукой.
- (−) Небольшой накладной расход на ведение записей (только для значимых решений).
